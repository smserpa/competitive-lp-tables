Easy Internationalization and Content Localization

Content is tied together with locales - if you update one, you must have all languages published

WHY? dotCMS requires a heavy reliance on Java, Tomcat, and their proprietary language Velocity. This means that CMS customization and updates can be difficult to manage. Additionally, code versioning may be managed externally in Github whereas Zesty.io handles all content and code versioning out of the box.

Zesty.io also handles locales differently in a way that separates all records - so if an English version of page A is ready to publish and the Spanish version is not, those can be managed separately. In dotCMS, all localized records are tied together which causes strained workflow for multilingual content implementations.
